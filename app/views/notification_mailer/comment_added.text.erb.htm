Lesson 40: Emailing Basics

Ultimately in our application, automated emails should be sent to the user who created a place after someone leaves a comment for it. In the next five lessons, we will cover various aspects of sending automated emails using Ruby on Rails. In this lesson, we will begin writing the first code to start sending emails from our application.

Many web applications send automated emails to users when certain actions happen within the application. Because sending email is such a common thing to do, Rails includes the ability to send emails in the framework.

The details of how emails are sent are in the Action Mailer Documentation.

For our web application, let's add the feature that has the user who created a place receive an automated email message as soon as a comment gets added on one of their places.

So if a user creates a place called Princess Pizza and then receives a comment from someone that says "I love Princess Pizza so, so much!" an automatic email should be sent letting the user know that a new comment for their Princess Pizza place was just added.

Trigger an Email Inside Our Web Application

If you look in the Action Mailer Documentation you can see an example of creating a mailer at section 2.1. Let's follow those steps that it outlines. Since our mailer is going to be all about sending out notifications we can call it NotificationMailer.

We can generate a mailer by running this command the terminal.

$ rails generate mailer NotificationMailer

If you go back to the Action Mailer documentation and scroll to section 2.1.2, it tells us the next step is to edit the created mailer and gives an example of how to do it.

First open up app/mailers/notification_mailer.rb and it should look like this:

class NotificationMailer < ApplicationMailer
end

We want to set up a "default from" so that appears our emails were sent from the following email "no-reply@nomsterapp.com". To do this we'll add the following code:

class NotificationMailer < ApplicationMailer
  default from: "no-reply@nomsterapp.com"
end

Inside this single mailer file we can describe multiple emails that we want to be able to send out. We can add a template for any particular email by adding a section for it, much like we do with our controllers.

Since our notification email is going to be about "added comments", I'm going to add a section in the mailer for it:

class NotificationMailer < ApplicationMailer
  default from: "no-reply@nomsterapp.com"

  def comment_added
  end
end

If we look back in the Action Mailer documentation, we can see that the mail method is what will actually send out our email.

Let's add this in and for now simply hard-code (meaning we won't use variables) all the values into our method to test if everything works. Make sure to enter your own email address in the "to" address.

class NotificationMailer < ApplicationMailer
  default from: "no-reply@nomsterapp.com"

  def comment_added
    mail(to: "my@email.com",
         subject: "A comment has been added to your place")
  end
end

Save the file.

Return to the action mailer documentation again and scroll down to 2.1.3. You can see that we need to add a view for the email content that will be sent out.

Here's how emails works. There are two types of emails you can get in your inbox. The first type, the HTML type, allows you to use bold, italics, links, and images, change fonts, and so on. This type allows us to send HTML content, similar to how our web pages are built. We should point out that email clients don't support all of the advanced features of web browsers, and typically you want to stick to basic HTML stuff in these emails (think bold, italic, links). If you want to have flashy emails, I suggest buying an email template, rather than building it yourself, because it can take hours and hours to get right. You can see some of the options if you google email template.

Also many mail clients don't allow HTML content, so it's important that we specify what we should send if an email client rejects the HTML version. Both emails should have the same message and information, but in the plain text emails you won't have the ability to add things like bold, italics and images.

Adding the Email Content: Creating the Two Types of Views

To start, let's add a file called comment_added.html.erb in app/views/notification_mailer/, and add some code that looks like this:

<p>Hey There!</p>
<p>
    Congrats!  <b>A comment has been added on Nomster</b>
</p>

Aloha,<br />
Ken

Save the file.

Scroll down in the Action Mailer documentation until you read:

Not all clients prefer HTML emails, and so sending both is best practice.

So let's do that too. Create a new file called app/views/notification_mailer/comment_added.text.erb, and make it look like this:

Hey There!

Congrats.  A comment has been added on Nomster

Aloha,
Ken

Then save this file.

Now let's test if our mailer actually sends out email. To do this jump into the rails console:

$ rails console

And when you're in the rails console run this line of code, it should trigger the mailer:

> NotificationMailer.comment_added.deliver
